Чек-лист для тестирования функции get_triangle_type 


Позитивные тесты (валидные треугольники) 


1. Равносторонний треугольник:
   • Вход: (3, 3, 3)
   • Ожидаемый результат: "equilateral"

2. Равнобедренный треугольник (a = b):
   • Вход: (5, 5, 8)
   • Ожидаемый результат: "isosceles"

3. Равнобедренный треугольник (a = c):
   • Вход: (7, 10, 7)
   • Ожидаемый результат: "isosceles"

4. Равнобедренный треугольник (b = c):
   • Вход: (12, 9, 9)
   • Ожидаемый результат: "isosceles"

5. Разносторонний треугольник:
   • Вход: (3, 4, 5)
   • Ожидаемый результат: "nonequilateral"

6. Разносторонний треугольник (порядок сторон не важен):
   • Вход: (5, 4, 3)
   • Ожидаемый результат: "nonequilateral"


Негативные тесты (невалидные треугольники)



1. Нулевая сторона:
   • Вход: (0, 4, 5)
   • Ожидаемая ошибка: IncorrectTriangleSides

2. Отрицательная сторона:
   • Вход: (3, -4, 5)
   • Ожидаемая ошибка: IncorrectTriangleSides

3. Сумма двух сторон равна третьей:
   • Вход: (1, 2, 3)
   • Ожидаемая ошибка: IncorrectTriangleSides

4. Сумма двух сторон меньше третьей:
   • Вход: (1, 1, 3)
   • Ожидаемая ошибка: IncorrectTriangleSides

5. Все стороны нулевые:
   • Вход: (0, 0, 0)
   • Ожидаемая ошибка: IncorrectTriangleSides

6. Нечисловые аргументы:
   • Вход: ("a", 2, 3)
   • Ожидаемая ошибка: TypeError (или IncorrectTriangleSides, в зависимости от реализации)
